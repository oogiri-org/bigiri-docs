## 1. サービス概要

bǃgǃrǃ は、オンラインで大喜利を遊ぶための Web アプリです。利用者は部屋を作成し、出題、回答、投票、結果発表を繰り返して遊びます。

- ブラウザーから利用できます。
- スマートフォンとデスクトップの両方で利用できます。
- 複数の部屋が同時に公開されます。
- パスワード付きの部屋を作成できます。
- 部屋は最終更新日時から 11.5 日間保持され、その後は自動終了します。
- 過去の結果の表示とログのダウンロード対象は、34.5 日以内の出題です。
- 対応ブラウザーは ECMAScript 2022 相当のモダンブラウザーです。
- 古い Internet Explorer、旧 Edge、古い Firefox、Safari、Chrome は対象外です。

## 2. 基本的な遊び方

最も基本的な流れは次の通りです。

1. トップ画面でサインインします。
2. 既存の部屋に入るか、新しい部屋を作成します。
3. 出題フェーズでお題を投稿します。
4. 回答フェーズで回答を投稿します。
5. 投票フェーズで気に入った回答に投票します。
6. 結果フェーズで順位、得票、コメントを確認します。
7. 次の出題へ進みます。

部屋の設定によって、出題できる人、回答できる人、投票できる人、制限時間、匿名表示、回答の表示タイミング、投票の表示タイミングなどが変わります。

## 3. サインインと利用者名

### 3.1 通常サインイン

通常サインインでは、任意の名前を入力して利用を開始します。名前の初期値は「お客さん」です。

名前は、部屋内での出題者名、回答者名、投票者名、コメント投稿者名として使われます。同じ名前の利用者がいる場合でも、内部的にはディスクリミネーターで区別されます。

### 3.2 Twitter（X）でサインイン

Twitter（X）アカウントを使ってサインインできます。

- 入室者一覧にソーシャルリンクを表示できます。
- ナビゲーションメニューや部屋一覧で X 関連のリンクが表示される場合があります。
- 入室者を見る画面でプロフィール画像が表示される場合があります。
- 通常サインインと比べて、未出題の部屋を複数作成できる扱いがあります。
- 画面遷移は外部サービス側の仕様に合わせて変更される場合があります。

### 3.3 Bluesky でサインイン

Bluesky アカウントによる OAuth サインインに対応しています。リリースノート上では試験稼動として導入されています。

- サインイン画面から Bluesky でサインインできます。
- トップ画面や部屋画面では、X の検索・投稿リンクと並んで Bluesky のリンクが表示される場合があります。
- 入室者一覧のソーシャルリンク表示にも利用できます。

### 3.4 サインオーバー

サインオーバーは、現在のサインイン状態を別のブラウザーや端末へ引き継ぐ機能です。

1. トップ画面または部屋画面のメニューからサインオーバーを開きます。
2. 表示された URL または QR コードを引き継ぎ先のブラウザーで開きます。
3. 引き継ぎ先では 1 時間以内にアクセスする必要があります。
4. URL をコピーするボタンも利用できます。

共有先を間違えると、サインイン状態を他人に渡すことになります。URL や QR コードの取り扱いには注意してください。

### 3.5 名前の変更

名前はサインアウトせずに変更できます。現在は「名前を変更する」という独立メニューではなく、画面上に表示されている自分の名前そのものがメニューとして動作します。

- トップ画面で名前を押すと、トップ画面で使う名前を変更できます。
- 部屋画面で名前を押すと、その部屋内で使う名前を変更できます。
- 部屋内で変更した名前は、その部屋の中だけに反映されます。
- 部屋画面の名前メニューから、入室者一覧に表示するソーシャルリンクの URL や表示有無も設定できます。

### 3.6 名前へのハッシュ値付加

名前に `#文字列` を付けると、`#` 以降が暗号学的ハッシュ値に変換され、8 文字の文字列として表示されます。

| 入力例 | 表示例 |
|:--|:--|
| `びぎりちゃん` | `びぎりちゃん` |
| `びぎりちゃん#abc` | `びぎりちゃん#DnbJh2JQ` |
| `風祭ビギリマナ春子#abc` | `風祭ビギリマナ春子#C2GkNEL6` |

この機能は、同名利用者の識別や、特定の相手にだけ同一人物性を伝えたい場合に使えます。

### 3.7 ディスクリミネーター

ディスクリミネーターは、入室者ごとに割り当てられる 8 文字の識別子です。入室者を見る画面などで表示され、同名利用者の区別に使われます。

ログのディスクリミネーター付き CSV では、名前の後ろに改行を挟んでディスクリミネーターが付加されます。名前にハッシュ値を付加している場合は、ハッシュ値つきの名前の後ろにディスクリミネーターが続きます。

## 4. トップ画面

トップ画面は、部屋一覧と主要メニューの入口です。

### 4.1 部屋一覧

トップ画面には公開中の部屋が一覧表示されます。

- 部屋名、作成者、入室者、出題日時などを確認できます。
- 部屋にアイコン画像が設定されている場合、一覧にも表示されます。
- 自分が入室している部屋では、入室者名の中で自分が判別できるように表示されます。
- 表示順は「作成順」と「更新順」から選べます。どちらも日時の降順です。
- 部屋名に関連する X や Bluesky の検索・投稿リンクが表示される場合があります。

### 4.2 更新情報

トップ画面では更新情報を確認できます。

- 更新情報の履歴はダイアログで表示できます。
- 更新情報では、過去の「標準化変量」表記が現在は「Zスコア」に統一されています。
- AI で試験的に生成されたユーザーマニュアルへのリンクが表示される場合があります。

### 4.3 メニュー

トップ画面のメニューから、名前変更、サインアウト、サインオーバー、部屋作成などを実行できます。

## 5. 部屋に入る

部屋は大喜利を行う単位です。トップ画面の部屋一覧から部屋を選ぶと、部屋画面へ移動します。

- パスワード付きの部屋では、入室時にパスワードが必要です。
- 部屋説明に書かれた URL は自動的にリンクになります。
- 部屋右下には、設定されているアイコン画像が表示されます。
- 部屋画面のメニューバーには、自分の名前、グループ名、カラースキーム、累計ユニコーンなどが表示されます。
- メニューバーのタブで、メイン画面、入室者を見る画面、グループを選ぶ画面、過去の結果画面、ログ画面などに移動できます。

## 6. 部屋を作成する

トップ画面から新しい部屋を作成できます。部屋作成者は、部屋名、説明、パスワード、アイコン画像、各種ゲーム設定を指定します。

### 6.1 基本項目

| 項目 | 内容 |
|:--|:--|
| 部屋名 | 部屋一覧や部屋画面に表示される名前です。 |
| 説明 | 部屋のルールや案内を書きます。URL は自動リンク化されます。 |
| パスワード | 設定すると入室時に入力が必要です。 |
| アイコン画像 | 部屋右下や部屋一覧に表示されます。容量制限があります。 |
| カラースキーム | 部屋のデフォルト配色をライトまたはダークから選べます。 |

### 6.2 部屋作成履歴

同じブラウザーで作成した部屋の設定を履歴として利用できます。

- 部屋名の横の履歴ボタンから過去の設定を表示できます。
- 読み込むボタンで、過去の部屋設定をフォームへ反映できます。
- 削除するボタンで、履歴から設定を削除できます。
- 同じ部屋名で作成した場合、履歴には最新の設定だけが残ります。
- アイコン画像も直近の設定値が初期値として引き継がれます。
- 履歴画面では部屋の作成日時を確認できます。

### 6.3 未出題部屋の制限

通常サインインでは、未出題のまま保持できる部屋数に制限があります。リリースノート上では、未出題の部屋作成は 1 つまでとされています。

Twitter（X）でサインインしている場合は、未出題のまま複数の部屋を作成できる扱いがあります。

## 7. 部屋設定の詳細

部屋設定はゲームの進行ルールを決めます。部屋作成後に入室者一覧で個別指定を行った場合、個別指定が初期設定より優先されます。

### 7.1 参加者制限

| 設定 | 内容 |
|:--|:--|
| 出題者制限 | 出題できる入室者を制限します。 |
| 回答者制限 | 回答できる入室者を制限します。 |
| 投票者制限 | 投票できる入室者を制限します。 |

制限には、制限なし、部屋作成者が指定した入室者のみ、Twitter（X）サインイン者のみといった選択肢があります。

### 7.2 出題関連

| 設定 | 内容 |
|:--|:--|
| 出題時間制限 | 出題フェーズの制限時間です。 |
| 出題者優先 | 前回の投票に参加した回答者のうち上位者などを優先できます。 |
| 出題優先時間 | 優先者だけが出題できる時間を設定できます。 |
| 出題画像制限 | 画像付き出題を許可するか制御します。 |
| 出題コメント制限 | 出題コメントを投稿できる人を制限します。 |

出題優先時間中は、優先される出題者の「出題する」ボタンが明滅します。部屋作成者のみ出題可能にする設定もあります。部屋作成者は、出題優先を途中で締め切れます。

### 7.3 回答関連

| 設定 | 内容 |
|:--|:--|
| 回答時間制限 | 回答フェーズの制限時間です。 |
| 回答シャッフル時間 | 指定時間内の回答を投票フェーズで入室者ごとにランダム順表示します。 |
| 回答者匿名 | 回答者名を表示するタイミングを制御します。 |
| 回答表示 | 回答を即時表示するか、一斉表示するかを選びます。 |
| 回答数制限 | 1 出題あたりの回答数を制限します。 |
| 回答描画制限 | 手描き線画を使った回答を許可するか制御します。 |
| 回答コメント制限 | 回答コメントを投稿できる人を制限します。 |

回答シャッフル時間を空にした場合、全回答がシャッフル対象になります。結果フェーズの順位表示は、このシャッフル設定とは別に計算されます。

### 7.4 投票関連

| 設定 | 内容 |
|:--|:--|
| 投票時間制限 | 投票フェーズの制限時間です。回答数に比例して時間を追加する設定があります。 |
| 投票者匿名 | 投票者名を常時非表示、締切後表示、記名などから制御します。 |
| 投票表示 | 投票を即時表示するか、一斉表示するかを選びます。 |
| 投票数制限（1 題毎） | 1 つの出題全体で使える票数を制限します。 |
| 投票数制限（1 回答毎） | 1 つの回答に入れられる票数を制限します。 |
| 投票グループ | グループ単位で投票対象を制御します。 |

投票数制限（1 題毎）には、制限を超えて投票できる「超過可」の設定があります。投票を見る画面では、投票数の合計の横に 1 題毎の投票数制限が表示されます。

### 7.5 コメント関連

コメントには、部屋コメント、出題コメント、回答コメントがあります。

- 部屋コメントは、部屋作成者が部屋上部に表示する案内として使えます。
- 出題コメントと回答コメントは、主に結果フェーズで投稿・表示されます。
- コメント投稿は部屋設定で制限できます。
- 回答コメント制限には、自回答のみコメント可能にする設定があります。

## 8. ゲームのフェーズ

各出題は 4 つのフェーズで進行します。

| フェーズ | 内容 |
|:--|:--|
| 出題フェーズ | お題を投稿します。 |
| 回答フェーズ | 回答を投稿します。 |
| 投票フェーズ | 回答に投票します。 |
| 結果フェーズ | 順位、得票、コメント、各種指標を確認します。 |

回答表示や投票表示が「一斉」の場合、他の人の回答や投票は所定のフェーズまでまとめて表示されます。制限時間がある場合は、画面上に残り時間や処理中の状態が表示されます。

## 9. 出題する

### 9.1 出題フォーム

出題はテキストで投稿します。現在の出題文字数制限は 1500 文字です。

- 長い出題では、途中に残りを展開するボタンが表示されることがあります。
- 出題には画像を添付できます。
- 部屋設定で出題画像が禁止されている場合、画像は使えません。
- 画像お題の本文の初期値は画像ファイル名になります。
- 出題画像の背景色は、ライト配色の背景色と同じ白です。
- 出題には出典欄があります。

他者の著作物や外部素材を使う場合は、出典欄に必要な情報を入力してください。

### 9.2 出題の送信

出題入力欄では、次のショートカットで送信できます。

| 操作 | キー |
|:--|:--|
| 入力内容を送信 | `Ctrl + Enter` |
| 入力内容を送信 | `Meta + Enter` |

Meta キーは Windows キーまたは Command キーを指します。

### 9.3 出題優先

部屋設定によっては、出題できる人が一定時間優先されます。

- 前回の投票に参加した回答者のうち上位者が優先される場合があります。
- 部屋作成者のみが出題できる時間を設定できます。
- 優先される人の「出題する」ボタンは明滅します。
- 優先時間は部屋作成者が途中で締め切れます。

## 10. 回答する

### 10.1 回答フォーム

回答はテキストで投稿します。現在の回答文字数制限は 1500 文字です。

- 長い回答では、途中に残りを展開するボタンが表示されることがあります。
- 回答数がリアルタイムで表示される場合があります。
- 直前に削除した回答がある場合、その内容が回答欄の初期値になる場合があります。
- 入力文字数を超過した場合はエラーメッセージが表示されます。

### 10.2 回答の送信

回答入力欄でも、出題と同じショートカットを使えます。

| 操作 | キー |
|:--|:--|
| 入力内容を送信 | `Ctrl + Enter` |
| 入力内容を送信 | `Meta + Enter` |
| ダイアログを閉じる | `Escape` |

### 10.3 回答の変更と削除

回答は変更または削除できる場合があります。

- 回答を変更すると、回答時刻も更新されます。
- 回答を削除した後、直前の内容が次回の入力欄に入る場合があります。
- 削除可否や表示タイミングは部屋設定やフェーズにより異なります。

### 10.4 回答描画

部屋設定で許可されている場合、手描き線画を回答に追加できます。回答描画は、テキストだけでは表現しにくい回答を投稿するための機能です。

## 11. 投票する

### 11.1 投票の基本

投票フェーズでは、表示された回答に対して票数を指定して投票します。

- 部屋設定により、1 題あたりの総票数が制限される場合があります。
- 部屋設定により、1 回答あたりの票数が制限される場合があります。
- 投票者匿名設定により、投票者名が非表示または締切後表示になる場合があります。
- 投票表示が一斉の場合、投票フェーズになってから投票できるよう統一されています。

### 11.2 投票数の操作

投票する画面では、投票数ボタンに初期フォーカスが設定されます。投票数ボタンにフォーカスがある状態で次の操作ができます。

| 操作 | キー |
|:--|:--|
| 投票数を増やす | `Space` |
| 投票数を減らす | `Delete` |
| 投票内容を送信 | `Enter` |

過去にはマウスドラッグで投票数を増減する機能がありましたが、現在は廃止されています。

### 11.3 0票投票

0 票を明示的に投票できます。現在は「投票する」ボタンの横にある `0` 票ボタンから、0 票のクイック投票を実行できます。

0 票投票は、回答を見たが票は入れないことを明示したい場合や、投票参加状態を残したい場合に使えます。

### 11.4 投票を見る画面

投票を見る画面では、自分が行った投票を確認できます。

- 投票数の合計を確認できます。
- 投票数制限（1 題毎）がある場合、合計の横に上限が表示されます。
- 回答ごとの投票は、投票数の多い順に表示されます。
- 投票数が同じ場合は投票順に表示されます。
- 投票フェーズでは、回答一覧の上部にも投票を見るボタンが表示されます。

以前は最後に投票した回答へ自動スクロールする機能がありましたが、現在は廃止されています。

## 12. 結果を見る

結果フェーズでは、順位、得票数、投票者、コメント、Zスコアに基づく装飾などを確認できます。

### 12.1 順位決定ルール

順位は得票数をもとに決まります。同票の場合は次の順で比較されます。

1. 回答者の投票数が多い回答を上位にします。
2. 回答者の投票数も同じ場合、回答時刻が早い回答を上位にします。

### 12.2 Zスコア

Zスコアは、回答の得票状況を相対的に表す指標です。過去の更新情報では「標準化変量」と表記されていましたが、現在は「Zスコア」に統一されています。

ユニコーンの算定には、回答者の投票数で補正した得票数が使われます。

### 12.3 ユニコーンとアイスクリーム

| 条件 | 表示 |
|:--|:--|
| Zスコア 1 以上 | 順位に下線が表示されます。 |
| Zスコア 2 以上 | ユニコーンが表示されます。さらに 1 上がるごとに 1 つ増えます。 |
| Zスコア -1 以下 | 順位に上線が表示されます。 |
| Zスコア -2 以下 | アイスクリームが表示されます。さらに 1 下がるごとに 1 つ増えます。 |

過去には Zスコア -2 以下の記号としてモアイ像が使われていましたが、現在はアイスクリームに変更されています。

### 12.4 累計ユニコーン

入室者一覧と部屋画面のメニューバーには、その部屋で獲得したユニコーンの累計が表示されます。入室者一覧ではユニコーン数による並び替えもできます。

## 13. コメントとリアクション

### 13.1 部屋コメント

部屋作成者は、部屋上部に表示するコメントを投稿できます。部屋コメントは部屋の案内、注意事項、進行メモに使えます。

部屋コメントを削除する機能は、最新 1 件を削除する動作です。

### 13.2 出題コメントと回答コメント

出題コメントと回答コメントは、結果フェーズや過去の結果画面で確認できます。

- 部屋作成者以外も投稿できる設定があります。
- 部屋設定で投稿権限を制限できます。
- 回答コメントには、自分の回答にのみコメントできる設定があります。
- 空文字列のコメントも許可される仕様があります。

### 13.3 クイックコメント

クイックコメントは、絵文字で手軽にリアクションする機能です。

- 回答コメントボタンの横にあるスマイルボタンから使います。
- ドロップダウンリストから絵文字を選ぶと、その絵文字がコメントとして投稿されます。
- 出題コメントにもクイックコメント機能があります。
- 絵文字の内容は更新で入れ替わることがあります。

## 14. グループ機能

グループは、部屋内の入室者を分類する機能です。大人数の部屋やチーム分けのある企画で、回答や投票の表示対象を整理できます。

### 14.1 グループの作成と管理

- 部屋作成者はグループを作成できます。
- グループは作成された部屋内でのみ有効です。
- グループ名は重複できます。
- グループ名は空文字列も許可される仕様です。
- グループには説明欄を設定できます。
- グループ作成画面には履歴機能があります。
- 同一グループ名で作成した場合、履歴には最新の設定だけが残ります。

### 14.2 グループの選択

グループが追加された部屋では、入室者はグループを選ぶ画面から自分のグループを選択できます。

- グループが追加された時点で未所属の入室者には、グループを選ぶ画面が表示されます。
- グループ追加後に入室した入室者にも、最初にグループを選ぶ画面が表示されます。
- 選んだグループが削除された場合も、グループを選ぶ画面が表示されます。
- 自分のグループは後から変更できます。

### 14.3 グループ表示

グループを使用している部屋では、各画面にグループ名が表示されます。

- メニューバーでは、自分の名前の前に自分のグループ名が表示されます。
- メイン画面と過去の結果画面では、回答者名の前にグループ名が表示されます。
- 入室者を見る画面では、入室者名の前にグループ名が表示されます。
- ログでは、回答者名カラムやグループ名カラムにグループ情報が出力されます。

### 14.4 投票グループ

投票グループは、投票フェーズで表示される回答をグループ単位で制御する設定です。

| 設定 | 投票フェーズで表示される回答 |
|:--|:--|
| 無し | 全体の回答が対象です。 |
| グループ内に投票 | 自分と同じグループの回答だけが表示されます。 |
| 有り（次グループに投票） | 自分のグループの次に作成されたグループの回答だけが表示されます。 |

自分のグループが最後の場合、「次グループ」は最初に作成されたグループになります。グループを選んでいない場合は、グループを選んでいない回答だけが表示されます。

投票グループを設定した場合、投票時間制限の「回答毎」に使われる回答数は、全体の回答数ではなく、表示対象の回答数が最も多いグループの回答数になります。グループが作成されていなければ、投票グループ設定がない場合と同様に動作します。

### 14.5 グループフィルター

結果フェーズと過去の結果画面では、グループ名を押すことで、そのグループの回答に絞り込めます。

## 15. 入室者一覧と管理

入室者を見る画面では、部屋にいる利用者の情報を確認できます。部屋作成者は、この画面から権限や強制退室に関する管理を行えます。

### 15.1 表示される情報

入室者一覧では、次の情報を確認できます。

- 名前
- ディスクリミネーター
- グループ名
- 入室者数
- その部屋で獲得した累計ユニコーン
- ソーシャルリンク
- Twitter（X）サインイン者のプロフィール画像
- 新規登録者マーク
- 受刑者状態

新規登録から 24 時間以内の入室者には、新規登録者マークが表示されます。このマークを押すと登録日時を確認できます。

### 15.2 入室者一覧の並び替え

入室者一覧では表示順を変更できます。

| 表示順 | 内容 |
|:--|:--|
| 先入順 | 入室が早い順です。 |
| 後入順 | 入室が遅い順です。 |
| 多ユニコーン順 | 累計ユニコーンが多い順です。 |
| 少ユニコーン順 | 累計ユニコーンが少ない順です。 |
| 昇名順 | 名前の Unicode 値の昇順です。 |
| 降名順 | 名前の Unicode 値の降順です。 |
| 昇 Di 順 | ディスクリミネーターの昇順です。 |
| 降 Di 順 | ディスクリミネーターの降順です。 |

### 15.3 権限指定

部屋作成者は、入室者一覧から入室者を出題者、回答者、投票者へ追加または削除できます。

個別指定がある場合、部屋作成画面で設定した出題者制限、回答者制限、投票者制限は、その項目について無視されます。

### 15.4 ソーシャルリンク設定

部屋画面のメニューバーで自分の名前を押すと、入室者一覧に表示するソーシャルリンクの URL や表示有無を設定できます。

以前は、部屋内で名前を変更している場合や名前にハッシュ値を付加している場合、ソーシャルリンクが表示されない仕様でした。現在、その自動非表示機能は廃止されています。

### 15.5 受刑者

受刑者は、強制退室対象として扱われる入室者です。

- 部屋作成者は入室者を受刑者へ追加または削除できます。
- 受刑者に追加された入室者は強制退室になります。
- 受刑者は入室者一覧で取り消し線つきで表示されます。
- 同一の発信元から出題、回答、投票、コメントを行う入室者も追加される場合があります。
- 不正なアクセスを行おうとした入室者は、自動的に受刑者へ追加される場合があります。

退室した出題者、回答者、投票者も、入室者一覧に含まれる場合があります。

## 16. 表示順・フィルター・フィーチャー

### 16.1 回答表示順

回答一覧では、フェーズや画面に応じて表示順を変更できます。

| 表示順 | 内容 |
|:--|:--|
| 設定順 | 部屋設定に基づく表示順です。 |
| 先着順 | 回答が早い順です。 |
| 後着順 | 回答が遅い順です。 |
| 上位順 | 順位が高い順です。 |
| 下位順 | 順位が低い順です。 |
| コ多順 | コメント数が多い順です。 |
| コ少順 | コメント数が少ない順です。 |

コ多順とコ少順は、結果フェーズと過去の結果画面で使えます。上位順と下位順は全フェーズで使える表示順として追加されています。

### 16.2 自分に関するフィルター

結果フェーズや過去の結果画面では、次のフィルターを利用できます。

| フィルター | 内容 |
|:--|:--|
| 自回答フィルター | 自分の回答だけを表示します。 |
| 自投票フィルター | 自分が投票した回答だけを表示します。 |
| 自コメントフィルター | 自分がコメントした回答だけを表示します。 |

投票フェーズでは、自投票フィルターと投票を見るボタンが機能的に近い役割を持ちます。自回答フィルターや自投票フィルターは、回答フィーチャー機能と併用できます。

### 16.3 フィーチャー

フィーチャーは、特定の回答だけを大きく表示する機能です。結果フェーズと過去の結果画面で使えます。

- 回答のフィーチャーボタンを押すと、その回答だけが表示されます。
- もう一度押すと通常表示に戻ります。
- 自回答フィルターや自投票フィルターと併用できます。

## 17. 過去の結果とログ

### 17.1 過去の結果

部屋の過去の結果画面では、過去 34.5 日以内の出題結果を参照できます。

- 結果フェーズと同じように順位やコメントを確認できます。
- 自回答、自投票、自コメント、グループなどのフィルターを利用できます。
- 表示順の変更やフィーチャー機能も利用できます。
- 直近の結果画面は廃止され、過去の結果画面に統合されています。

### 17.2 ログ（CSV）

ログ画面では、部屋の記録を CSV としてダウンロードできます。

- ダウンロードには入室している必要があります。
- URL を選択してからダウンロードが始まる方式です。
- ダウンロード時にサーバーが処理中であることが表示されます。
- グループ名を含む CSV に対応しています。
- 9 番目のカラムにグループ名が出力されます。
- ディスクリミネーター付き CSV では、名前の後ろに改行を挟んでディスクリミネーターが出力されます。

### 17.3 Atom フィード

Atom フィードが用意されています。フィードには部屋作成者名も含まれます。部屋の更新を外部リーダーなどで追跡したい場合に使えます。

## 18. 表示設定とショートカット

### 18.1 カラースキーム

部屋画面のメニューバーからカラースキームを切り替えられます。

| 設定 | 内容 |
|:--|:--|
| オート | ブラウザーや OS の設定に従います。 |
| ライト | ライト配色に固定します。 |
| ダーク | ダーク配色に固定します。 |

メニューバーでの選択は保存されません。部屋作成時には、部屋のデフォルトのカラースキームをライトまたはダークから設定できます。

### 18.2 ショートカット一覧

| 操作 | キーまたは方法 |

## 19. 退室・削除・自動終了

### 19.1 退室

入室者は、専用の退室する画面から部屋を退室できます。

退室後も、過去に出題、回答、投票を行った入室者は、入室者一覧や結果の整合性のために表示され続ける場合があります。

### 19.2 部屋の削除

部屋作成者は部屋を削除できます。削除後に必要な結果やログを確認できなくなる可能性があるため、必要な場合は事前に過去の結果や CSV ログを確認してください。

### 19.3 自動終了

部屋は最終更新日時から 11.5 日間が経過すると自動終了します。過去の結果とログは 34.5 日以内の出題が対象です。

## 20. 変更・廃止された主な機能

現在は提供されていない機能や、現行仕様と異なる過去の挙動をまとめます。

- 「標準化変量」という表記は「Zスコア」に変更されました。
- 「名前を変更する」専用メニューは廃止され、名前そのものがメニューになりました。
- 部屋内で名前を変更している場合や、名前にハッシュ値を付加している場合にソーシャルリンクを自動非表示にする機能は廃止されました。
- 投票を見る画面で最後に投票した回答へ自動スクロールする機能は廃止されました。
- 回答ヘッダーのクリックでその回答へスクロールする機能は廃止されました。
- マウスドラッグで投票数を増減する機能は廃止されました。
- Zスコア -2 以下で表示される記号は、モアイ像からアイスクリームへ変更されました。
- Discord 通知機能は廃止されました。
- bǃgǃrǃ 公式 Discord サーバーは廃止されました。
- 称号機能は廃止されました。
- 名前へのプリフィックスやサフィックスを設定する機能は廃止されました。
- reCAPTCHA による通常サインイン時の確認は廃止されました。
- 直近の結果画面は過去の結果画面に統合されました。
- グループの初期グループ設定機能は廃止され、必要な入室者にはグループを選ぶ画面を表示する方式になりました。
- マイストックドンは廃止されました。
- 部屋設定の URL 項目は廃止され、説明内 URL の自動リンク化に変更されました。
- 古いブラウザー向けの後方互換性は段階的に終了しています。
