## 1. サービス概要

bǃgǃrǃ は、オンラインで大喜利を楽しむための Web アプリケーションです。ユーザーは「部屋」を作って出題し、入室した他のユーザーが回答（ボケ）を投稿し、それらに投票し合って順位を決めるというサイクルを繰り返して遊びます。

- 日本語 UI、ブラウザから利用可能。
- スマートフォンからもデスクトップからも利用可能。
- 同時に複数の部屋が公開され、誰でも入室できます（パスワード付きの部屋も作れます）。
- 部屋は最終更新日時から **11.5 日間** 保持された後、自動的に終了します。
- 過去の結果は **34.5 日以内** に作成・完了された出題についてのみ参照できます。
- 対応ブラウザーは ECMAScript® 2022 に準拠したモダンブラウザーです。Internet Explorer や旧 Edge、古い Firefox / Safari / Chrome は非対応です。
- クリエイティブ・コモンズ・ライセンスの掲示はトップ画面から確認できます。

## 2. サインインとアカウント

bǃgǃrǃ では、部屋に入室したり出題・回答・投票を行うために「サインイン」が必要です。サインイン方法は複数あり、サインイン後に表示される「名前」があなたの識別子となります。

### 2.1 通常サインイン

サインイン画面で好きな名前を入力してサインインします。名前の初期値は「お客さん」です。

### 2.2 Sign in with Twitter（X アカウントでのサインイン）

Twitter（現 X）アカウントでサインインできます。以下の特徴があります。

- 入室者一覧に表示される名前から、当該 X アカウントへのリンクが張られます（ソーシャルリンク設定による）。
- ナビゲーションメニューに X リンクが表示されます。
- 入室者を見る画面でプロフィール画像が表示されます。
- 未出題のまま複数の部屋を同時に作成できます（通常サインインではできない場合があります）。
- 過去に Twitter 側の仕様変更に伴い、画面遷移や挙動が暫定的に変更されることがあります。最新の仕様は画面の案内に従ってください。

### 2.3 Bluesky でサインイン（OAuth）

Bluesky アカウントを使った OAuth サインインに対応しています（試験稼動）。

- サインイン画面で「Bluesky でサインイン」ボタンを押して認可を行います。
- サインイン後、入室者一覧や部屋一覧には X リンクと並んで Bluesky リンクが表示されます。

### 2.4 サインオーバー（セッション引き継ぎ）

別のブラウザ・端末にサインイン状態を引き継ぐための機能です。

1. トップ画面または部屋画面のメニューから「サインオーバー」を選びます。
2. 表示される URL または QR コードを、引き継ぎ先のブラウザで **1 時間以内** に開きます。
3. サインオーバー画面には「URL をコピーする」ボタンがあり、コピーしたリンクを引き継ぎ先に送ることでも引き継げます。

### 2.5 名前のハッシュ値付加

名前に `#文字列` を付けてサインインすると、`#` 以降の部分が暗号学的ハッシュ関数で 8 文字の文字列に変換されて表示されます。

- 例: `びぎりちゃん#abc` → `びぎりちゃん#DnbJh2JQ`
- 例: `風祭ビギリマナ春子#abc` → `風祭ビギリマナ春子#C2GkNEL6`

これにより、同名の入室者を区別したり、同一人物であることを特定の相手にだけ共有したりできます。

### 2.6 ディスクリミネーター

入室者それぞれに自動的に割り当てられる 8 文字の英数字の識別子です。入室者を見る画面の右側に表示され、同名他人との区別に使われます。CSV ログでは名前の直後に改行を挟んで付加されます。

### 2.7 名前の変更

- トップ画面のメニューから名前を変更できます。名前そのものがメニューとなっており、押すと変更フォームが開きます。
- 部屋画面のメニューから名前を変更すると、その部屋でのみ名前が変わります。
- サインアウトすることなく名前を変更できます。

### 2.8 退室

部屋からの退室は、専用の「退室する画面」から行えます。

## 3. トップ画面

トップ画面は公開されている部屋の一覧が表示されるエントリーポイントです。

### 3.1 部屋一覧

- 各部屋のアイコン画像、部屋名、作成者、出題日時、入室者名などが表示されます。
- 自分が入室している部屋の入室者欄では自分が識別できるように表示されます。
- 表示順はドロップダウンリストから「作成順」「更新順」（いずれも日時の降順）を選べます。
- 部屋名をハッシュタグとして X または Bluesky で検索・投稿するソーシャルリンクが表示されます。

### 3.2 更新情報

- トップ画面には更新情報（リリースノート）のダイジェストが表示されます。
- 「更新情報を見る」ボタンからダイアログで全履歴を確認できます。
- 更新情報のリンクから、AI で試験的に生成された「bǃgǃrǃ ユーザーマニュアル」を参照できます（本書はそのマニュアルの最新版です）。

### 3.3 メインメニュー

- 「名前」ボタン: 名前の変更が可能。名前そのものがメニューになっており、押すと変更フォームが開きます。
- 「サインアウト」「サインオーバー」「部屋を作成する」などの操作が行えます。

## 4. 部屋（ルーム）の仕組み

「部屋」は一連の大喜利ゲームを行う場です。1 つの部屋は複数の出題を繰り返して進行します。

- 部屋作成者が部屋の設定を決めます。
- 入室者は「出題者」「回答者」「投票者」になれます（部屋作成者による制限あり）。
- 部屋はパスワードで保護できます。
- 部屋には説明欄があり、説明内に書いた URL は自動的にリンクになります。
- 部屋にはアイコン画像を設定でき、部屋画面の右下に常時表示されます。
- アイコン画像には容量制限があります。
- 部屋存続時間は最終更新日時から 11.5 日間。期限を過ぎると自動的に終了します。

### 4.1 メニューバー

部屋画面の上部にはメニューバーが表示されます。

- 自分の名前（押すとソーシャルリンクの URL や表示設定の編集ができる）。
- 自分のグループ名（グループ機能が使われている場合）。
- タブ切り替えでメイン画面・入室者を見る画面・グループを選ぶ画面・過去の結果画面・ログ画面などに移動できます。タブ切り替え時には非共通部分までスクロールします。
- 部屋のカラースキーム切り替えボタン（オート/ライト/ダーク）。
- その部屋で獲得したユニコーン🦄の累計。

## 5. ゲームのフェーズ

各出題は以下の 4 フェーズで進行します。

| フェーズ | 概要 |
|:--|:--|
| 出題フェーズ | 出題者が題を投稿するのを待つフェーズ。 |
| 回答フェーズ | 回答者がボケを投稿できるフェーズ。時間制限・回答数制限あり。 |
| 投票フェーズ | 投票者が回答に投票するフェーズ。時間制限・投票数制限あり。 |
| 結果フェーズ | 順位・得票数・コメントを閲覧できる最終フェーズ。次の出題も可能。 |

- 回答表示は部屋設定により「即時」または「一斉」（投票フェーズ開始時にまとめて表示）を選べます。
- 投票表示も同様に「即時」または「一斉」が選べます。
- 一斉設定の投票は、投票フェーズになってから投票できるよう統一されています。
- 残り時間はメニューバー等にタイマー表示されます。

## 6. 部屋を作成する

トップ画面の「部屋を作成する」から部屋作成画面を開きます。

### 6.1 基本項目

- **部屋名**: 必須。一覧で識別される名称。
- **説明**: 部屋の紹介文。説明に書かれた URL は自動的にリンクになります。
- **パスワード**: 任意。設定すると入室時にパスワード入力が必要。
- **アイコン画像**: 任意。部屋右下に常時表示される画像（容量制限あり）。
- **カラースキーム**: デフォルトのカラースキームを「ライト」「ダーク」から選択。

### 6.2 部屋作成の履歴機能

- 部屋名欄の横の「履歴」ボタンを押すと、同じブラウザで作成した部屋の一覧が開きます。
- 「読み込む」ボタンでその部屋の設定を現在の作成フォームに読み込めます。
- 「削除する」ボタンで履歴から設定を削除できます。
- 同一の部屋名で作成すると、履歴には最新の設定のみが残ります。
- アイコン画像についても直近の設定値が初期値として引き継がれます。
- 履歴画面では部屋の作成日時が表示されます。

### 6.3 未出題の部屋の作成制限

- 通常サインイン時は、未出題のまま複数の部屋を作ることはできません（1 つまで）。
- Sign in with Twitter でサインインしている場合、未出題でも複数の部屋を作成できます。

## 7. 部屋の設定項目一覧

以下は部屋作成画面で指定できる代表的な設定項目です。

### 7.1 参加者制限系

| 設定 | 内容 |
|:--|:--|
| 出題者制限（初期値） | 無し / 有り / 有り（Sign in with Twitter） |
| 回答者制限（初期値） | 無し / 有り / 有り（Sign in with Twitter） |
| 投票者制限（初期値） | 無し / 有り / 有り（Sign in with Twitter） |

「有り」に設定すると入室者一覧で部屋作成者が個別に設定した入室者のみが対象になります。「有り（Sign in with Twitter）」の場合は X でサインインしている入室者に限定されます。入室者一覧で個別設定を行っている場合、部屋作成画面で行った設定はその項目について無視されます。

### 7.2 出題関連

| 設定 | 内容 |
|:--|:--|
| 出題時間制限 | 出題フェーズの時間制限。 |
| 出題者優先 | 前回の投票フェーズで 1 位となった投題者が優先的に出題できる設定。投票していない回答者は出題優先の対象外になります。 |
| 出題優先時間 | 無し / 有り（秒数）/ 有り（部屋作成者） の三択。指定時間内は優先者以外の出題が禁止されます。部屋作成者のみ出題可にすることも可能です。部屋作成者は出題優先を途中で締め切れます。優先される出題者の「出題する」ボタンは明滅します。 |
| 出題画像制限 | 出題の画像使用を禁止できます。 |
| 出題コメント制限 | 無制限 / 部屋作成者のみ の二択。 |

### 7.3 回答関連

| 設定 | 内容 |
|:--|:--|
| 回答時間制限 | 回答フェーズの時間制限。 |
| 回答シャッフル時間 | 指定時間（秒）内の回答は投票フェーズで入室者毎にランダムな順番で表示されます。空指定の場合は全回答がシャッフル対象になります。結果フェーズの並びはこの設定と無関係です。 |
| 回答者匿名 | 常に非表示 / 締切後表示 などの選択。 |
| 回答表示 | 即時 / 一斉（投票フェーズ開始時にまとめて表示）。 |
| 回答数制限 | 1 出題あたりの回答数上限。 |
| 回答描画制限 | 回答への手描き線画の可否。 |
| 回答コメント制限 | 無制限 / 自回答のみ / 部屋作成者のみ。 |

### 7.4 投票関連

| 設定 | 内容 |
|:--|:--|
| 投票時間制限 | 投票フェーズの時間制限。回答数に比例して追加する設定も可能。 |
| 投票者匿名 | 常に非表示 / 締切後表示 / 記名。 |
| 投票表示 | 即時 / 一斉。 |
| 投票数制限（1 題毎） | 無し / 有り / 有り（超過可）。「有り（超過可）」は制限に達しても投票画面で超過できるが、投票するボタンは投票一覧ボタンに変わります。 |
| 投票数制限（1 回答毎） | 無し / 有り。 |

### 7.5 グループ関連

| 設定 | 内容 |
|:--|:--|
| 投票グループ | 無し / グループ内に投票 / 有り（次グループに投票）。設定により投票フェーズで表示される回答が、自分と同じグループ・自分の次に作成されたグループ・グループ未選択者のいずれかに限定されます。 |

## 8. 出題する

### 8.1 出題フォーム

- 出題はテキストで行います（最大 1500 文字）。
- 150 文字を超える出題には閾値を超えた位置に残文字数ボタン「•••」が表示され、押すと残りが展開されます。
- 画像を添付できます（出題画像制限が無しの場合）。
- 画像お題の本文のデフォルト値は画像のファイル名になります。
- 出題画像の背景色は白（ライトスキーム背景色と同一）です。

### 8.2 出典の入力

- 出題には「出典」欄があります。著作物を用いる場合は必ず適切な出典を入力してください。

### 8.3 出題の送信

- 出題入力欄では以下のキー操作で送信できます。
  - Ctrl + 改行
  - メタキー（Windows キー / Command キー）+ 改行
- 競合が発生した場合（同時に複数人が出題しようとした場合）は、システムが適切に処理します。

## 9. 回答する

### 9.1 回答フォーム

- テキスト回答が基本（最大 1500 文字）。
- 150 文字を超える部分には「•••」ボタンが表示されます（出題と同じ仕様）。
- 送信ショートカット: Ctrl + 改行 / メタ + 改行。
- Escape キーでダイアログウィンドウを閉じられます。
- 直前に削除された回答がある場合は、それがデフォルトとして回答欄に設定されます。
- リアルタイムで回答数が表示されます。

### 9.2 回答描画（手描き線画）

- 回答に手描きの線画を追加できます（回答描画制限が「無し」の場合）。
- スマートフォン・タブレットでもサンプルレートが最適化されており、滑らかに描けます。

### 9.3 回答を変更する / 削除する

- 「回答を変更する」ボタンで既存の回答を変更できます。回答時刻も更新されます。
- 回答を削除することもできますが、部屋設定によっては削除不可になる場合があります。
- 直前に削除した回答の内容は次回の回答欄の初期値として残ります。

## 10. 投票する

### 10.1 投票の基本

- 回答を見て、ボタンから投票を行います。
- 複数の回答に投票できる場合と、1 回答につき複数票を入れられる場合があります（部屋設定による）。

### 10.2 0 票投票

- 投票フェーズで「0 票」を明示的に投票できます。
- 「投票する」ボタンの横の「👍0」ボタン（0 票クイック投票）を押すと、即座に 0 票を投票できます。
- 0 票を投票した後の投票では、投票を見る画面で末尾にスクロールします。

### 10.3 票数を増減する

投票画面の投票数ボタン（👍 数字）は以下の方法で値を変えられます。

- ボタンにフォーカスがある状態で：
  - スペースキー: 増票
  - デリートキー: 減票
  - 改行キー: 送信
- 投票画面では投票数ボタンに初期フォーカスが設定されます。

### 10.4 投票を変更する・削除する

- 投票を一旦削除することなく投票数を減らせる「投票を変更する」ボタンがあります。
- 投票を削除することもできます（部屋設定による）。
- 投票数制限に達した場合、「投票一覧」ボタンが表示されます。
- 投票数制限（1 題毎）が「有り（超過可）」の投票では、投票を見る画面で末尾にスクロールします。

### 10.5 投票を見る

- 投票フェーズで「投票を見る」ボタンを押すと、これまでに自分が行った投票の一覧が表示されます。
- 投票を見る画面には投票数の合計の横に投票数制限（1 題毎）も表示されます。
- 各回答ごとの投票の表示順は投票数の降順、同数の投票は投票順になります。
- 投票フェーズでは「投票を見る」ボタンが回答一覧の上部にも表示されます（自投票フィルターと同等）。

## 11. コメントする

### 11.1 部屋コメント

- 部屋作成者が部屋画面の上部に常時表示されるコメントを投稿できます。
- 部屋コメントを削除する機能は、最新 1 件のみを削除する動作になっています（全件削除ではない）。

### 11.2 出題コメント・回答コメント

- 結果フェーズで部屋作成者以外でも投稿・表示できます（削除は不可）。
- 部屋設定により投稿制限が可能（無制限 / 部屋作成者のみ / 自回答のみ）。
- 過去の結果画面にも出題コメント・回答コメントが表示されます。
- 空文字列のコメントも許可されます。
- 出題コメントにもクイックコメント（リアクション）機能があります。

### 11.3 クイックコメント（リアクション）

- 結果フェーズの各回答の「回答コメント」ボタンの横にある 🙂 ボタンを押し、ドロップダウンから絵文字を選ぶとその絵文字をコメントとして投稿できます。
- 現在 8 種類の絵文字が用意されており、適宜入れ替えられることがあります。
- 絵文字フォントは Google Fonts Noto Color Emoji に統一されています。

## 12. グループ機能

グループは入室者をまとめるための仕組みです。

### 12.1 グループの作成・管理

- 部屋作成者が「グループを選ぶ画面」からグループの作成・削除を行えます。
- グループ名は重複可・空文字列も許可されます。
- グループは作成された部屋内でのみ有効です。
- グループには説明欄を付けられます。
- グループ作成画面には履歴機能があり、履歴ボタンから過去のグループ設定を読み込めます（同じブラウザで作成したグループ）。
- 同一のグループ名で作成した場合、履歴には最新の設定のみが残されます。

### 12.2 グループへの加入

- グループが作成されると、グループに入っていない入室者にはグループを選ぶ画面が表示されます。
- グループ作成後に入室した入室者にも、最初にグループを選ぶ画面が表示されます。
- 選んだグループが削除された場合、再度グループを選ぶ画面が表示されます。
- 入室者は「グループを選ぶ画面」から自分のグループをいつでも変更できます。

### 12.3 グループ分離

- グループが追加された部屋ではグループ分離が常に有効になります。
- 回答・投票フェーズで自分が入っているグループ以外の回答は表示されません。
- 投票時間制限の「回答毎×」の計算には、全体の回答数ではなく最も回答数の多いグループの回答数が用いられます。
- グループ分離が有効でもグループを作成していなければ無効時と同じ動作です。

### 12.4 投票グループ

部屋設定「投票グループ」により投票フェーズの可視範囲が制御されます。

- **無し**: 全ての回答が対象。
- **グループ内に投票**: 自分と同じグループの回答だけが表示される。投票時間制限の「回答毎×」は表示された回答数が最も多いグループの回答数が採用されます。
- **有り（次グループに投票）**: 自分のグループの直後に作成されたグループの回答だけが表示される。自分が最後のグループなら最初に作成されたグループの回答。グループ未選択者にはグループ未選択者の回答のみが表示される。

### 12.5 グループの表示

- ナビゲーションバーでは自分の名前の前に自分が入っているグループ名が表示されます。
- メイン画面・過去の結果画面・入室者を見る画面で各入室者名の前にグループ名が表示されます。
- 部屋設定の投票グループが「無し」の場合は、部屋画面でグループ毎の回答数は表示されません。
- CSV ログの回答者名カラムには、グループ名・改行・回答者名の順で出力されます。CSV ログの 9 番目のカラムにもグループ名が出力されます。

### 12.6 グループフィルター

- 結果フェーズおよび過去の結果画面でグループ名を押すと、そのグループの回答のみに絞り込めます。

## 13. 入室者管理

### 13.1 入室者を見る画面

- 入室者一覧には全入室者の情報が表示されます。
  - 名前・ディスクリミネーター
  - その部屋で獲得したユニコーン累計🦄
  - プロフィール画像（X サインイン時）
  - グループ名
  - 新規登録者マーク🌱（24 時間以内に登録した入室者）
  - 受刑者（取り消し線付き）
  - 入室者数
- 表示順はドロップダウンから選択できます。
  - 先入順 / 後入順（入室時刻）
  - 多🦄順 / 少🦄順（ユニコーン獲得数）
  - 昇名順 / 降名順（名前の Unicode 値）
  - 昇 Di 順 / 降 Di 順（ディスクリミネーターの Unicode 値）
- 退室した出題者・回答者・投票者も一覧に含まれます。
- 新規登録者マーク🌱をクリックすると登録日時が表示されます。
- 入室者の名前をクリックするとトークン（入室者一覧の右側の文字列）が表示されます。
- 部屋作成者が幅の狭いブラウザで入室者を見る画面を表示させた場合、操作ボタン群を開閉できます。

### 13.2 ソーシャルリンクの設定

- 部屋画面のメニューバーで自分の名前を押すと、入室者一覧に表示するソーシャルリンクの URL や表示の有無を設定できます。
- X・Bluesky リンクが同等に表示されます。
- かつての「部屋内で名前を変更している場合や名前にハッシュ値を付加している場合に入室者一覧でソーシャルリンクを表示しない機能」は廃止されました（2026 年 4 月）。

### 13.3 出題者・回答者・投票者の指定

- 部屋作成者は入室者一覧から個別に出題者・回答者・投票者を設定・解除できます。
- この個別設定は、部屋作成画面での「出題者制限」「回答者制限」「投票者制限」設定より優先されます。

### 13.4 受刑者（強制退室）

- 部屋作成者は入室者一覧から入室者を受刑者に追加でき、追加された入室者は強制退室になります。
- 受刑者は入室者一覧に取り消し線付きで表示されます。
- 入室者を受刑者に追加した場合、同一の発信元から出題・回答・投票・コメントする入室者も自動的に追加されます。
- 不正なアクセスで出題・回答・投票・コメントを行おうとした入室者は自動的に受刑者に追加されます。

### 13.5 部屋内での名前変更

- 部屋画面のメニューから名前変更が可能です。
- ここで変更した名前はその部屋内でのみ有効です。

## 14. 順位・ユニコーン・Zスコア

結果フェーズでの各回答の得票数と順位は、「回答者の投票数で補正した得票数」に基づき計算されます。

> **用語に関するお知らせ**: バージョン 1.132.0（2026-04-25）より、これまで「標準化変量」と表記していた指標を **「Z スコア」** に統一しました。仕様自体は変わりません。

### 14.1 順位決定ルール

- 得票数が多い順に順位が決まります。
- 得票数が同じ場合は回答者の投票数が多い方が上位です。
- 投票数も同じ場合は回答が早い方が上位です。

### 14.2 ユニコーン🦄

- 補正得票数の Z スコアが 2 以上の回答には順位の横にユニコーン🦄が表示されます。
- Z スコアが 1 上がるごとに🦄が 1 つ増えます。

### 14.3 アイスクリーム🍦

- 補正得票数の Z スコアが -2 以下の回答には順位の横にアイスクリーム🍦が表示されます（以前はモアイ像🗿でした）。
- Z スコアが 1 下がるごとに🍦が 1 つ増えます。

### 14.4 順位装飾（下線・上線）

- Z スコアが 1 以上: 順位に下線。
- Z スコアが -1 以下: 順位に上線。

### 14.5 自回答・自投票の表示

- 部屋設定の回答者名表示・投票者名表示の設定値にかかわらず、自回答の回答者名および自投票の投票者名は常に表示されます。

### 14.6 累計ユニコーン

- 入室者を見る画面・部屋のメニューバーに、その部屋で獲得したユニコーンの累計が表示されます。

### 15.4 フィーチャー機能（回答の個別表示）

- 結果フェーズおよび過去の結果画面で 👁 を押すとその回答のみを全画面的に表示します。

- 自回答・自投票フィルターと併用できます。

## 17. ログ（CSV）のダウンロード

部屋のログ画面から CSV ファイルをダウンロードできます。

### 17.1 ダウンロード手順

1. 部屋のログ（CSV）画面を開きます。
2. ダウンロードしたい URL を選択すると、サーバーで処理を開始します（処理中はその旨が表示されます）。
3. ダウンロードの際には入室している必要があります。

### 17.2 ファイル形式

- 基本の CSV には出題・回答・投票・コメント等が出力されます。
- 9 番目のカラムに回答者のグループ名が出力されます。
- 回答者名カラムの先頭にグループ名と改行が追加されます。
- ディスクリミネーター付き CSV も選択可能です。名前の後に改行を挟んでディスクリミネーターが付加されます。名前にハッシュ値を付加している場合は「名前#ハッシュ値 改行 ディスクリミネーター」の形式になります。

## 22. 廃止・変更された機能（履歴）

現在は提供していませんが、過去のバージョンで利用できた機能や大きな変更履歴の一部を示します。

- **部屋画面からの「名前を変更する」メニュー**: 専用メニューは廃止され、名前そのものをメニューとして動作するように統一されました（2026-04-04）。

- **reCAPTCHA**: 通常サインイン時の reCAPTCHA 確認は廃止されました（2021-08-16）。
- **初期グループの設定機能**: 廃止され、グループ追加後の入室者にはグループを選ぶ画面が表示されるようになりました（2024-06-18）。

- **サインアウト時の最短時間制限**: 廃止されました（2020-12-14）。
- **部屋設定 URL 項目**: 部屋設定の URL を廃止し、説明内に書かれた URL が自動でリンクになるようにしました（2018-09-01）。
- **モアイ像🗿**: Z スコア -2 以下の回答に表示されていたモアイ像🗿は、アイスクリーム🍦に変更されました（2024-01-19）。

## 24. 付録 B: 用語集

- **ボケ**: 出題に対する回答。
- **ユニコーン🦄**: 優れた回答に付与される記号。Z スコアに基づく。
- **ディスクリミネーター**: 入室者ごとに自動的に割り振られる 8 文字の英数字 ID。
- **Z スコア**: 得票数の偏差を集団内の標準偏差で割った値（標準化変量とも呼ばれる）。バージョン 1.132.0 より用語を「Z スコア」に統一。
- **サインオーバー**: サインイン状態を別のブラウザ・端末に引き継ぐ機能。
- **受刑者**: 部屋作成者が指定、または不正行為により自動的に強制退室させられた入室者。
- **グループ分離**: グループ間で回答・投票を分離する仕組み。
- **投票グループ**: 投票フェーズで表示対象を自分のグループ・次のグループ等に限定する設定。
- **部屋存続時間**: 部屋が自動終了するまでの時間（最終更新日時から 11.5 日）。
- **過去の結果**: 34.5 日以内の出題結果を参照できる画面。
- **出題優先時間**: 前回 1 位の投票者または部屋作成者だけが出題できる時間。
- **回答シャッフル時間**: 投票フェーズで入室者毎にランダム順で表示される回答の対象時間。
