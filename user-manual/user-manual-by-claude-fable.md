## 1. サービス概要

bǃgǃrǃ（ビギリ）は、オンラインで大喜利を楽しむための Web アプリケーションです（サブタイトル: For Beautiful Oogiri Life）。ユーザーは「部屋」を作成または入室し、「出題 → 回答 → 投票 → 結果」というサイクルを繰り返して遊びます。

- ブラウザーだけで利用でき、スマートフォンにも対応しています。
- 部屋は最終更新日時から **11.5 日間** 存続し、その後自動的に終了します。
- 過去の結果の参照とログ（CSV）のダウンロードは **34.5 日以内** の出題が対象です。
- トップ画面にはクリエイティブ・コモンズ・ライセンスが掲示されています。
- トップ画面の更新情報には、AI で試験的に生成された「bǃgǃrǃ ユーザーマニュアル」へのリンクがあります。

## 2. サインインとアカウント

部屋への入室、出題・回答・投票・コメントにはサインインが必要です。

### 2.1 通常サインイン

名前を入力してサインインします。名前の初期値は「お客さん」です。サインイン時には reCAPTCHA による確認が行われます。

### 2.2 名前へのハッシュ値の付加（なりすまし防止）

名前の末尾に `#文字列` を付けると、その部分が暗号学的ハッシュ関数値（8文字）に変換されて表示されます。同じ文字列からは常に同じハッシュ値が得られるため、本人確認の手段として使えます。

```
例:
びぎりちゃん          → びぎりちゃん
びぎりちゃん#abc      → びぎりちゃん#DnbJh2JQ
風祭ビギリマナ春子#abc → 風祭ビギリマナ春子#C2GkNEL6
```

### 2.3 ソーシャルサインイン

#### Twitter（X）でサインイン

Sign in with Twitter によるサインインができます。

- 入室者一覧の名前に X アカウントへのリンクが表示されます（表示の有無は設定可能。2.5 節参照）。
- ナビゲーションメニューに Twitter リンクが表示されます。
- 入室者を見る画面でプロフィール画像が表示されます。画像をロードできない場合はデフォルトの画像が表示されます。
- 未出題のまま複数の部屋を作成できます（通常サインインでは未出題の部屋は 1 つまで）。
- 現在、Twitter でサインインする際の画面遷移には暫定的な措置が適用されています。この措置は予告なく終了する場合があります。

#### Bluesky でサインイン

Bluesky アカウントによる OAuth サインインができます（試験稼動）。サインイン画面の「Bluesky でサインイン」ボタンから認可を行います。

### 2.4 サインオーバー（別ブラウザーへの引き継ぎ）

サインイン状態を別のブラウザー（別の端末）へ引き継ぐ機能です。

1. トップ画面または部屋画面のメニューから「サインオーバー」を選びます。
2. 表示される URL または QR コードを、引き継ぎ先のブラウザーで **1 時間以内** に開きます。
3. サインオーバー画面には URL をコピーするボタンがあります。

### 2.5 名前の変更とソーシャルリンク設定

- トップ画面・部屋画面のメニューに表示されている **名前自体がメニュー** になっており、押すことで名前を変更できます。サインアウトは不要です。
- 部屋内で名前を変更すると、その部屋でのみ変更後の名前が使われます。
- 部屋画面のメニューの名前を押すと、入室者一覧に表示するソーシャルリンクの URL や表示の有無を設定できます。部屋内で名前を変更している場合や名前にハッシュ値を付加している場合でも、ソーシャルリンクは設定に従って表示されます。

### 2.6 退室

退室する画面から部屋を退室できます。なお、退室した出題者・回答者・投票者も入室者一覧には含まれます。

## 3. トップ画面

トップ画面には公開中の部屋の一覧と各種情報が表示されます。

- **部屋一覧**: 部屋名・説明・アイコン画像・出題日時・入室者名が表示されます。入室者名では自分が判別できるように表示されます。
- **表示順**: ドロップダウンリストから「作成順」・「更新順」（いずれも日時の降順）を選べます。
- **ソーシャルリンク**: 部屋名をハッシュタグとして検索・投稿できる Twitter リンクと、同等の Bluesky リンクが表示されます。
- **更新情報**: 更新情報を見るボタンを押すと、更新履歴がダイアログウィンドウで表示されます。
- **Atom フィード**: 部屋の新着を Atom フィードで購読できます。フィードには部屋作成者名が含まれます。

## 4. 部屋の仕組みとゲームの流れ

### 4.1 フェーズ

部屋の中では 1 つの出題（お題）について、次のフェーズが順に進行します。

| フェーズ | 内容 |
|---|---|
| 出題 | 出題者がお題（文章・画像）を投稿します。 |
| 回答 | 入室者がお題に対する回答を投稿します。 |
| 投票 | 入室者が面白いと思った回答に投票します。 |
| 結果 | 得票数に基づく順位とコメントが表示されます。 |

結果フェーズの後、次の出題によって新しいサイクルが始まります。

### 4.2 部屋作成者の権限

部屋作成者は以下の操作ができます。

- 部屋設定の決定（作成時）
- 部屋コメントの投稿・削除
- 回答・投票の手動締め切り、出題優先の締め切り
- 入室者一覧での出題者・回答者・投票者の指定
- 受刑者（強制退室者）の追加・削除
- グループの作成・削除

### 4.3 部屋の存続期間

部屋は最終更新日時から 11.5 日間が経過すると終了します。記録を残したい場合はログ（CSV）をダウンロードしてください（17 章参照）。

## 5. 部屋を作成する

トップ画面から部屋を作成できます。部屋名・説明・各種設定（6 章参照）・アイコン画像を指定します。

- **説明内の URL** は自動的にリンクになります。
- **アイコン画像**: 部屋の右下に常に表示される画像を設定できます。容量制限があります。
- **履歴機能**: 部屋名の横の履歴ボタンを押すと、同じブラウザーで過去に作成した部屋の一覧が表示されます。「読み込む」ボタンでその設定を読み込み、「削除する」ボタンで履歴から削除できます。同一の部屋名で部屋を作成した場合、履歴には最新の設定のみが残ります。履歴では部屋の作成日時も確認できます。アイコン画像を含め、直近の設定値が次回作成時の初期値になります。
- 通常サインインの場合、未出題のままの部屋は 1 つまでしか作成できません。Twitter でサインインしている場合は未出題のまま複数の部屋を作成できます。

## 6. 部屋の設定項目一覧

部屋を作成する画面で設定できる項目です。

### 6.1 参加者の制限

| 項目 | 設定値 | 説明 |
|---|---|---|
| 出題者制限（初期値） | 無し / 有り / 有り（ソーシャルサインイン） | 出題できる入室者を制限します。 |
| 回答者制限（初期値） | 無し / 有り / 有り（ソーシャルサインイン） | 回答できる入室者を制限します。 |
| 投票者制限（初期値） | 無し / 有り / 有り（ソーシャルサインイン） | 投票できる入室者を制限します。 |

これらは「初期値」であり、部屋作成後に入室者一覧画面で個別に出題者・回答者・投票者を指定できます。入室者一覧画面で設定を行った場合、その項目については部屋作成時の設定は無視されます。

### 6.2 出題に関する設定

| 項目 | 説明 |
|---|---|
| 出題画像制限 | お題への画像の使用を禁止できます。 |
| 出題優先時間 | 「有り（投票者内1位）」では、結果発表後の指定時間、前回の投票者の中で 1 位だった入室者だけが出題できます。「有り（部屋作成者）」では指定時間、部屋作成者だけが出題できます。優先時間中は優先される出題者の「出題する」ボタンが明滅します。部屋作成者は出題優先を途中で締め切ることができます。 |

### 6.3 回答に関する設定

| 項目 | 説明 |
|---|---|
| 回答時間制限 | 回答フェーズの制限時間を設定します。 |
| 回答者匿名 | 回答者名を非表示にします。ただし自回答の回答者名は常に本人に表示されます。 |
| 回答表示 | 「即時」では回答が投稿され次第表示され、「一斉」では締め切り後にまとめて表示されます。 |
| 回答シャッフル時間 | 「有り（秒）」に設定すると、指定時間内の回答は投票フェーズで入室者毎にランダムな順番で表示されます。秒数を空にすると全ての回答がシャッフル対象になります。結果フェーズの順位はこの設定の影響を受けません。 |

### 6.4 投票に関する設定

| 項目 | 説明 |
|---|---|
| 投票時間制限 | 投票フェーズの制限時間を設定します。「回答毎×秒」を設定すると回答数に比例した時間が加算されます。投票グループ使用時は、全体の回答数ではなく表示された回答数が最も多いグループの回答数が用いられます。 |
| 投票者匿名 | 投票者名を非表示（常に非表示または締め切り後表示）にします。ただし自投票の投票者名は常に本人に表示されます。 |
| 投票表示 | 「即時」では投票が即座に反映され、「一斉」では投票フェーズになってから投票でき、締め切り後にまとめて表示されます。 |
| 投票数制限（1題毎） | 1 つのお題に投票できる総票数を制限します。「有り（超過可）」では、制限に達すると投票するボタンが投票一覧ボタンに変わりますが、投票画面では制限を超過して投票できます。 |
| 投票数制限（回答毎） | 1 つの回答に投票できる票数を制限します。 |
| 自回答投票 | 自分の回答への投票の可否を設定します。 |
| 投票グループ | グループ機能と連動した投票範囲の制御です（12 章参照）。 |

### 6.5 コメント・表示に関する設定

| 項目 | 説明 |
|---|---|
| 出題コメント制限 | 制限無し、または部屋作成者のみに制限できます。 |
| 回答コメント制限 | 制限無し、または部屋作成者のみに制限できます。 |
| 回答描画 | 回答への手描き線画の使用を制限できます。 |
| カラースキーム | 部屋のデフォルトのカラースキームを「ライト」・「ダーク」から設定できます。 |

## 7. 出題する

回答・投票の終了後（または部屋作成直後）、出題者は「出題する」ボタンからお題を投稿できます。

- **文字数**: 最大 1500 文字。150 文字を越える部分は折りたたまれ、「残りを表すボタン」を押すと表示されます。10 行を越えた位置にも同様のボタンが表示されます。
- **画像**: お題に画像を付けられます（部屋設定で禁止可能）。本文を省略した場合は画像のファイル名が本文の初期値になります。出題画像の背景色は白です。
- **出典**: お題に出典を入力・表示できます。お題に著作物を用いる場合は適切な出典を入力してください。
- **送信**: 入力欄で Ctrl+改行キー、またはメタキー（Windows キー・Command キー）+改行キーで送信できます。
- **出題優先時間**: 部屋設定により、結果発表後の一定時間は特定の入室者（前回投票者中の 1 位、または部屋作成者）だけが出題できます。優先される出題者の「出題する」ボタンは明滅します。

---

## 8. 回答する

回答フェーズでは、回答者がお題への回答を投稿します。

- **文字数**: 最大 1500 文字。150 文字（または 10 行）を越える部分は折りたたまれ、ボタンで展開できます。
- **回答の変更**: 「回答を変更する」ボタンで投稿済みの回答を変更できます。変更すると回答時刻も更新されます。
- **回答の削除と復元**: 回答を削除した直後に再回答する場合、削除した内容が回答欄の初期値として設定されます。
- **回答描画**: 回答に手描きの線画を追加できます（部屋設定で制限可能）。
- **回答数の表示**: 回答数はリアルタイムで表示されます。投票グループが「無し」の場合、グループ毎の回答数は表示されません。
- **送信**: Ctrl+改行キーまたはメタキー+改行キーで送信できます。

## 9. 投票する

投票フェーズでは、投票者が回答へ投票します。

### 9.1 投票の操作

- 回答の「投票する」ボタンを押すと投票画面が開きます。
- 投票画面では投票数ボタンで票数を指定して投票します。投票数ボタンには初期フォーカスが設定されており、**スペースキーで増票、デリートキーで減票、改行キーで送信** ができます。
- **0 票の投票** も可能です。「投票する」ボタンの横の「👍0」ボタン（0票クイック投票）を押すと、投票画面を開かずに 0 票を投票できます。
- 投票後も「投票を変更する」ボタンで、投票を削除することなく票数を増減できます。

### 9.2 投票の確認

- **投票一覧ボタン**: 自分のそれまでの投票を一覧表示し、削除できます。投票数制限を使い切った場合の調整に便利です。一覧にはお題と投票数の合計が表示されます。
- **投票を見るボタン**: 回答一覧の上部にもあり、自分が投票した回答のみを表示します（自投票フィルターと同等）。
- 投票を見る画面では、投票数の合計の横に投票数制限（1題毎）が表示されます。
- 回答毎の投票は投票数の降順で表示されます（同数の場合は投票順）。

### 9.3 投票数の制限

部屋設定により、1 題毎・回答毎の投票数制限、自回答への投票可否、投票時間制限が適用されます（6.4 節参照）。

## 10. 結果と順位

投票の締め切り後、結果フェーズで順位が表示されます。

### 10.1 順位の決定

- 順位は得票数に基づきます。
- 得票数が同じ回答は、**回答者自身の投票数が多い方** が上位になります。それも同じ場合は回答が早い方が上位です。

### 10.2 ユニコーンとアイスクリーム（Zスコア表示)

回答の補正得票数（回答者の投票数で補正した得票数）の Z スコアに応じて、順位に装飾が付きます。

| 条件 | 表示 |
|---|---|
| Z スコア +1 以上 | 順位に下線 |
| Z スコア +2 以上 | ユニコーン 🦄（さらに 1 上がる毎に 1 つ増加） |
| Z スコア −1 以下 | 順位に上線 |
| Z スコア −2 以下 | アイスクリーム 🍦（さらに 1 下がる毎に 1 つ増加） |

- ユニコーンの算定は回答者の投票数で補正されます（回答するだけでなく投票にも参加することが重要です）。
- その部屋で獲得したユニコーンの累計は、部屋画面のメニューバーと入室者一覧に表示されます。

## 11. コメント機能

### 11.1 部屋コメント

部屋作成者はいつでも部屋トップページの上部にコメントを表示できます。削除すると最新の 1 件が削除されます。

### 11.2 出題コメント・回答コメント

結果フェーズでは、部屋作成者以外の入室者もお題や回答にコメントを投稿できます（削除はできません）。投稿は部屋設定により部屋作成者のみに制限できます。空文字列のコメントも投稿できます。

### 11.3 クイックコメント（リアクション）

回答コメントおよび出題コメントでは、コメントボタンの横のスマイルボタンを押し、ドロップダウンリストから絵文字（8 種類）を選ぶだけでコメントできます。絵文字の種類は随時入れ替えられます。

---

## 12. グループ機能

入室者をまとめる「グループ」を使うと、チーム戦のような遊び方ができます。グループは作成された部屋内でのみ有効です。

### 12.1 グループの作成と参加

- 部屋の「グループを選ぶ」ボタンからグループを選ぶ画面を開けます。
- **部屋作成者** はこの画面からグループの作成・削除を行えます。グループ名は重複でき、空文字列も許されます。グループには説明欄もあります。
- **履歴機能**: グループ名の横の履歴ボタンから、同じブラウザーで作成したグループの設定を読み込み・削除できます。同一グループ名では最新の設定のみが残ります。
- 部屋作成者がグループを追加すると、グループに入っていない入室者には自動的にグループを選ぶ画面が表示されます。グループ追加後に入室した入室者や、選んでいたグループが削除された入室者にも表示されます。
- 自分のグループはグループを選ぶ画面からいつでも変更できます。

### 12.2 グループの表示

- ナビゲーションバーで自分の名前の前に自分のグループ名が表示されます。
- 回答者名・入室者名の前にもグループ名が表示されます。
- ログ（CSV）の回答者名カラムにはグループ名と改行が前置され、9 番目のカラムにもグループ名が出力されます。
- 結果フェーズおよび過去の結果画面でグループ名を押すと、グループフィルター（13 章参照）が使えます。

### 12.3 グループ分離と投票グループ

部屋作成者がグループを追加すると、回答・投票フェーズでは自分が入っているグループ以外の回答者の回答は表示されなくなります（グループ分離）。

さらに部屋設定の「投票グループ」により、投票フェーズで表示される回答の範囲を制御できます。

| 設定値 | 投票フェーズで表示される回答 |
|---|---|
| グループ内に投票 | 自分と同じグループの回答のみ |
| 有り（次グループに投票） | 自分のグループの直後に作成されたグループの回答のみ。自分のグループが最後の場合は最初に作成されたグループ。グループを選んでいない場合はグループを選んでいない回答のみ。 |

- 投票グループを設定した場合、投票時間制限の「回答毎×」には表示された回答数が最も多いグループの回答数が採用されます。
- 投票グループを設定してもグループを作成しなければ、設定が無い場合と同様に動作します。

## 14. 入室者の管理

### 14.1 入室者を見る画面

部屋の入室者一覧では以下が確認できます。

部屋作成者は入室者一覧から以下の操作ができます（幅の狭い画面では操作ボタン群を開閉できます）。

- **出題者・回答者・投票者の指定**: 入室者を出題者・回答者・投票者へ追加・削除できます。指定を行うと、部屋作成時の制限設定（初期値）はその項目について無視されます。
- **受刑者の追加・削除**: 受刑者に追加された入室者は強制退室となり、入室者一覧に取り消し線付きで表示されます。受刑者を追加すると、同一の発信元から出題・回答・投票・コメントする入室者も自動的に追加されます。また、不正なアクセスによって出題・回答・投票・コメントを行おうとした入室者は自動的に受刑者に追加されます。
